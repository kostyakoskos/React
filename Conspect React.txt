Компонетный подход. Параметры передаются через this.
DOM Document Object Model- представление пользовательского интерфейса в приложении. При каждом изменении UI Dom также обновляется для отображения этих изменений.
Virtual DOM. В React для каждого настоящего элемента существует соответствующий виртульный элемент. VDOM содержит что и Real DOM, но не может влиять на то, что отбражается на экране. Монипуляции с RDOM являются 
медленными. 
За счет чего VDOM быстрее работает? Когда в UI добавляются новые элементы, VDOM строит новое дерево. Каждый элемент - узел дерева. При изменении состояния любого элемента создается новое дерево. Затем это дерево
сравнивается со старым. После чего вычисляется наиболее эффективный сктод внесения изменений в RDOM. Цель вычислений- минимизация количества операций с RDOM. Все надо делать с VDOM.
React использует паттерн наблюдатель(GoF) и реагирует на изменение состояния. Каждая часть является компонентой, каждый компонент имеет состояние. После построения VDOM, React обновляет соответствующие элементы в RDOM.
2. React использует механизм пакетного обновления RDOM. Обновления отправлются в виде пакетов, а не отдельного обновления при изменении состояния. 
3. Эффективный поиск различий в дереве.
При каких-то изменениях, перестраивается virtual dom, а потом эти маленькие измениния(изменить текст на кнопке и тд) вносятся в real dom. В RDOM работа очень дорогостоящая. Новый VDOM сравнивается со старым, а потом
вычисляется способ за минимальное число шагов(минимальные затраты) достичь изменений в RDOM.
!Отнсится к работе без ключей! Пусть есть 4 article. Если мы удаляем article 4, то в RDOM удалится article4. А если надо удалить article1? Тогда в RDOM меняем у article1 text и title на article2 text title и тд.!
С ключами удалится просто article 1 и все.

Babel - интрумент, чтобы превращать javascript синтаксис в jsx.

Из jsx невозможно вернуть 2 элемента рядом.
Прежде чем jsx попадет в браузер, его надо скомпилировать. 
В jsx обязательно закрывать октрытый тег.
<div /> равноценно <div></div>

<Article> и <article>. В чем разница? Большая и маленькая буква. Article это переменная. article просто элемент. Компилятор будет пытавться отрендорить его название.
Все мои функции обязательно с большой буквы!
